# Security And Privacy

This repository contains reusable playbook templates only.

Do not commit:

- customer data;
- patient data;
- student records;
- booking records;
- call transcripts;
- phone numbers;
- private emails;
- API keys;
- production webhook URLs.

## Sensitive Industries

For clinics, admissions, legal, finance, housing, or emergency service workflows, treat the playbook as an intake aid only. Operators must check their local legal, privacy, consent, medical, telecom, and recordkeeping requirements.

## Reporting Issues

If a playbook encourages unsafe collection, unsafe claims, or excessive personal data, open an issue with the playbook name and the risky field/section.
